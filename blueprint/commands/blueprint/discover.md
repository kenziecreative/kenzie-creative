Run the `blueprint-discover` skill and follow its steps exactly.

Sweep the user's recurring work into a thin Process Inventory — candidates in their own words, none
rated for automation — then recommend a small first portfolio to capture. This is for when the user
can't yet name their processes; if they already know which process they want to model, point them at
`/blueprint:capture` instead.
