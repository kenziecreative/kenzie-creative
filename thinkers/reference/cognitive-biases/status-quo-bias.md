# Status Quo Bias

> The preference for the current state of affairs, and the tendency to see any change from that baseline as a loss.

## Description

Status quo bias is the preference for keeping the current state of affairs and treating change from that baseline as a loss, risk, or burden.

The current state feels natural because it is already in place. Alternatives have to prove themselves, while the status quo often gets a free pass.

## How It Works

The status quo benefits from familiarity, default effects, loss aversion, switching costs, and ambiguity avoidance.

Change requires attention and accountability. Staying put can feel safer, even when the current situation is quietly costly.

## Recognition Cues

- The current option is not evaluated as actively as alternatives.
- Change is treated as risky while staying put is treated as neutral.
- "We have always done it this way" carries weight by itself.
- Switching costs are counted but ongoing costs are ignored.
- You ask whether to change, but not whether you would choose the current state from scratch.

## Signature Phrasing

- "I prefer things as they are"
- "If it ain't broke, don't fix it"
- "The devil you know is better"
- "Change is risky"
- "Let's not rock the boat"

## Ask Yourself

> If we were starting from scratch today, would we choose the current arrangement?

## Common False Positives

Keeping the status quo can be wise. Change has real costs, risks, coordination burdens, and unintended consequences.

The bias appears when familiarity is mistaken for merit or when the costs of the current state are treated as invisible.

## What Helps

- Make the current state defend itself against alternatives.
- Count the cost of staying, not only the cost of switching.
- Run small reversible experiments where change feels too large.
- Ask what you would choose if no option were already installed.
- Separate implementation pain from long-term value.

## Evidence Status

Evidence status: well-supported in behavioral decision research, especially around defaults, choice architecture, and loss framing.
