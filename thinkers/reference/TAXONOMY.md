# Thinkers Toolkit Taxonomy

This taxonomy defines the lenses used by the corpus. The same real-world event can contain several patterns at once, but each entry should have a primary job: name a kind of distortion, argument move, influence move, coercive pattern, or strategic tool.

The purpose of the taxonomy is not to make the world neat. It is to help a user ask the right next question.

## Core Types

### Cognitive Bias

An automatic or semi-automatic pattern in perception, memory, judgment, attention, or motivation.

Use this type when the pattern mainly describes how a mind tends to distort reality without needing conscious intent.

Useful questions:

- What is the mind over-weighting or under-weighting?
- What feels true because of how it is perceived, remembered, framed, or emotionally loaded?
- What would help the person check their own judgment?

Not every bad argument is a cognitive bias. A bias lives in the judging mind; an argument may exploit that bias.

### Logical Fallacy

A flaw in inference: the conclusion does not follow from the premises, or the argument relies on an invalid reasoning structure.

Use this type when the pattern mainly describes why an argument is not logically sound.

Useful questions:

- What claim is being made?
- What premise is supposed to support it?
- Where does the reasoning fail?

Logical fallacies can be used innocently or strategically. The type names the reasoning flaw, not the speaker's intent.

### Rhetorical Fallacy

A persuasive argument move that evades scrutiny, shifts attention, or makes weak reasoning feel stronger.

Use this type when the pattern mainly describes how a speaker moves an audience away from evaluating the actual claim.

Useful questions:

- What is the audience being made to feel or notice instead of the argument?
- What changed in the frame, burden, target, or standard?
- What question would return the exchange to the actual claim?

Rhetorical fallacies sit between logic and persuasion: they may contain a reasoning flaw, but their power comes from presentation.

### Persuasion Tactic

An influence move that shapes perception, desire, trust, urgency, identity, or choice.

Use this type when the pattern can be ethical or unethical depending on truthfulness, transparency, consent, and proportionality.

Useful questions:

- What response is this tactic trying to create?
- What legitimate version of this tactic exists?
- Where does persuasion become manipulation?

Persuasion tactics should usually include a "legitimate cousin" or ethical boundary, because influence is not inherently coercive.

### Bad-Faith Move

A debate or accountability-avoidance move that pretends to participate in honest exchange while making honest exchange harder.

Use this type when the pattern mainly appears in argument, public discourse, meetings, or conflict where one party evades the burden of clarity, evidence, or accountability.

Useful questions:

- What work is the move avoiding?
- What burden is being shifted?
- What would restore the conditions of a fair exchange?

Bad faith is a behavioral pattern, not a permanent identity label. The corpus should name the move, not diagnose the person.

### Manipulation Tactic

A coercive or covert influence pattern that pressures a person toward compliance, dependence, self-doubt, silence, or distorted choice.

Use this type when the pattern primarily operates through asymmetry: hidden intent, emotional leverage, information control, relational pressure, fear, guilt, ambiguity, or isolation.

Useful questions:

- What autonomy is being constrained?
- What information, emotion, relationship, or dependency is being used as leverage?
- What would the honest version of the underlying need look like?

Manipulation entries should be especially careful about false positives. Many behaviors have benign versions. The entry must help the user distinguish pattern from single incident, harm from discomfort, and coercion from ordinary conflict.

### Strategy

A deliberate tool for decision-making, negotiation, planning, conflict, cooperation, ethics, or systems thinking.

Use this type when the concept is primarily something a person can apply on purpose to understand or shape a situation.

Useful questions:

- What problem does this strategy solve?
- What mechanism makes it work?
- What are the conditions for using it well?
- When does it become unethical, brittle, or counterproductive?

Strategy entries should not read like advice to dominate people. They should teach discernment: when to use the tool, when to avoid it, and what human cost it may carry.

Strategies are not all the same kind of thing. Some are applied practices, some are game-theory models, some are ethical frames, and some are descriptive social patterns. They share a type because users apply them as thinking tools, but their entries should not all use the same internal structure.

- Applied practices should teach use conditions, backfire modes, and decision prompts.
- Game-theory models should teach players, incentives, payoffs, equilibrium logic, and translation limits.
- Ethical frames should teach the core moral test and where it breaks down.
- Descriptive social patterns should teach formation, sustaining forces, recognition cues, and interventions.

## Overlap Rules

Some concepts can appear under more than one type. That is acceptable only when each entry provides a different lens.

Examples:

- Anchoring can be a cognitive bias when someone is unconsciously influenced by the first number, and a negotiation strategy when someone deliberately sets the first number.
- Social proof can be a heuristic, a persuasion tactic, or a fallacious appeal depending on whether it is being used as weak evidence, influence design, or flawed reasoning.
- Sunk cost can be a decision bias and a fallacy in argument. The bias entry should help someone notice the internal pull; the fallacy entry should show why "we already spent so much" does not justify continuing.

When a concept appears in multiple places, each entry should include a short "Lens Note" or "Not This" section that points to the related lens.

## Type Selection Test

When adding or revising an entry, ask:

1. Is the main issue inside someone's judgment? Use `cognitive_bias`.
2. Is the main issue invalid reasoning? Use `logical_fallacy`.
3. Is the main issue persuasive presentation that evades scrutiny? Use `rhetorical_fallacy`.
4. Is the main issue influence that may be legitimate or questionable? Use `persuasion_tactic`.
5. Is the main issue dishonest argument behavior or accountability avoidance? Use `bad_faith_move`.
6. Is the main issue coercive or covert interpersonal control? Use `manipulation_tactic`.
7. Is the main issue a deliberate tool for acting, deciding, or understanding a system? Use `strategy`.

If more than one answer fits, pick the type that best serves the user's next action and cross-link the rest.

## Labeling Discipline

The corpus should help people become more perceptive without becoming more paranoid.

Avoid turning every uncomfortable interaction into manipulation, every disagreement into bad faith, every emotion into irrationality, or every repeated behavior into pathology. Entries should name patterns clearly while preserving ordinary explanations: stress, misunderstanding, personality differences, incomplete information, memory limits, and honest disagreement.

A good entry leaves the user with sharper perception and more agency. It should not leave them with a weaponized vocabulary for condemning everyone around them.
