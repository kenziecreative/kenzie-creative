# The Post Hoc Fallacy

> The logical fallacy of assuming that because one event followed another, the first event caused the second.

## Description

The post hoc fallacy, short for post hoc ergo propter hoc, assumes that because one event happened before another, the first event caused the second.

Sequence can be evidence, but it is not proof. Causes usually precede effects, yet many things precede an effect without causing it.

This error is common in personal habits, business decisions, politics, medicine, and performance analysis because people naturally search for the cause of a change.

## Flawed Move

The flawed move is treating timing as enough to establish causation.

Common versions include:

- "We changed the process, then revenue improved, so the process caused it."
- "I wore this shirt and we won."
- "The new leader arrived and morale rose."
- "I took the supplement and felt better the next week."

## Why It Persuades

The mind wants causal closure. When an event changes, the most visible preceding event becomes an easy explanation.

Post hoc reasoning also flatters agency. It is satisfying to believe that a chosen action produced the result, especially when the result was positive.

## What It Avoids

The fallacy avoids causal testing:

- What else changed at the same time?
- Would the effect have happened anyway?
- Is there a plausible mechanism?
- Do comparison cases show the same relationship?
- Can the result be replicated?

## Common False Positives

Temporal order matters. If a proposed cause happened after the effect, it cannot be the cause. If it happened before, it may be relevant evidence.

The fallacy appears when sequence alone carries the argument. Stronger causal claims need mechanisms, controls, comparisons, repeated observations, or intervention evidence.

## How to Respond

Ask for the causal bridge between the events.

- "What else could explain the change?"
- "What mechanism connects the first event to the second?"
- "Did the same thing happen in comparable cases?"
- "How would we test whether this caused that?"

In decision reviews, separate timeline from causality: first list what changed, then evaluate which changes have evidence of impact.

## Decision Prompt

> Am I treating sequence as a clue to investigate, or as proof of cause?
