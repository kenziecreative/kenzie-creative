# Completeness and Richness Plan

This plan turns the corpus from a broad reference library into a consistently useful thinking toolkit.

## Goal

Every entry should help someone understand human behavior in motion: their own thinking, other people's communication, interpersonal influence, group dynamics, and decision-making under uncertainty.

The target is not academic exhaustiveness. The target is practical richness:

- A user can recognize the pattern in real situations.
- A user can distinguish it from nearby patterns and false positives.
- A user can see the mechanism underneath it.
- A user can choose a cleaner response.
- A user can check whether they are doing the pattern themselves.

## Batch Execution Queue

This queue is the operating plan for reworking the corpus without needing to renegotiate the next step after every batch.

### Operating Rhythm

For each batch:

1. Read the target entries and nearby related entries where needed.
2. Rewrite each entry using the right shape from `EDITORIAL-STANDARD.md`.
3. Replace generic related links with specific relationships.
4. Update `reference/INDEX.md` and the relevant folder `README.md`.
5. Run a link/shape verification pass.
6. Report what changed, any judgment calls, and the next queued batch.
7. Continue to the next batch unless the user redirects.

Each batch should be small enough to preserve quality. Eight to twelve entries is the normal range; fewer when entries are conceptually dense.

## Phase 1: Establish The Standard

Status: started.

Deliverables:

- `reference/TAXONOMY.md`
- `reference/EDITORIAL-STANDARD.md`
- This roadmap
- A small exemplar set across pattern types

Acceptance criteria:

- The type definitions are clear enough to classify new entries.
- The editorial standard is specific enough to rewrite entries without inventing a new shape each time.
- The exemplar entries show the desired richness in practice.

## Phase 2: Normalize Strategies

Why first: strategies are the broadest section and currently the thinnest. Many share generic related-pattern lists, empty contexts, and no severity/ethical framing.

Work:

- Add richer strategy sections, using the right sub-shape for the concept rather than forcing one template everywhere.
- Replace generic related links with specific ones.
- Add contexts where useful.
- Split or cross-link duplicate lenses such as confirmation bias, hindsight bias, sunk cost, anchoring, and social proof.
- Flag strategy entries that are defensive, cooperative, competitive, coercive, ethical, or decision-oriented.

Priority clusters:

1. Decision tools: opportunity cost, OODA loop, scenario analysis, backward induction.
2. Negotiation tools: BATNA, ZOPA, anchoring, logrolling, splitting the difference, deadline pressure.
3. Game-theory models: Nash equilibrium, Prisoner's Dilemma, Stag Hunt, dominant strategy, minimax, mixed strategies, War of Attrition, zero-sum/non-zero-sum.
4. Ethics and restraint: Golden Rule, veil of ignorance, categorical imperative, utilitarianism, boundary setting.
5. Dark/competitive strategies: deception, brinkmanship, honey trap, supremacy dynamics, crazy person strategy.
6. Descriptive social patterns: groupthink, herd behavior, Abilene paradox, Peter principle, tragedy of the commons.

Acceptance criteria:

- Strategy entries no longer read as generic advice cards.
- Each strategy uses a shape appropriate to its job: applied tool, game-theory model, ethical frame, or descriptive social pattern.
- Applied strategies teach when not to use them.
- Game-theory entries explain the model setup and the limits of translating the model into human life.
- Related links explain the local conceptual neighborhood.

## Phase 3: Enrich Persuasion and Bad-Faith Entries

Why second: these entries are close to real-world communication and need strong ethical boundaries.

Work:

- Add legitimate-cousin sections to persuasion tactics.
- Add "when it becomes manipulative" sections where appropriate.
- Add "what it avoids" sections to bad-faith moves.
- Strengthen response guidance without turning the entry into a debate script.
- Add false-positive guards.

Priority entries:

- Future faking
- Card stacking
- Strategic vagueness
- Manufactured certainty
- Appeal to rivalry
- Gish gallop
- Sealioning
- Kafka trap
- Whataboutism
- Tone policing

Acceptance criteria:

- Users can distinguish influence from coercion.
- Users can respond without escalating or over-labeling.

## Phase 4: Enrich Biases and Fallacies

Why third: the entries are usable but need better boundaries, evidence notes, and self-application.

Work:

- Add recognition cues beyond signature phrasing.
- Add common false positives.
- Add "what helps" or "how to respond" sections.
- Add evidence status notes for contested psychology concepts.
- Clarify overlapping concepts across types.
- For cognitive biases, include the inside-view feel, strengthening conditions, and a debiasing move where useful.

Priority entries:

- Confirmation bias
- Sunk cost fallacy
- Availability bias / availability heuristic
- Hindsight bias
- Framing effect
- Ego depletion
- Backfire effect
- Dunning-Kruger effect
- False dilemma
- Appeal to emotion

Acceptance criteria:

- The entries help users check themselves, not just label others.
- Contested or commonly misused concepts carry a caveat.

## Phase 7: Audit and Maintain

Work:

- Check for broken related links.
- Check for duplicate concepts without lens notes.
- Check for generic related lists.
- Check for empty contexts where contexts would help.
- Check for entries under 50 lines after enrichment candidates are selected.
- Add a lightweight review checklist for future entries.

Acceptance criteria:

- No missing related targets.
- No large batch of entries shares an unexplained generic related list.
- New entries can be reviewed against the editorial standard.

## Working Principle

The corpus should make users more perceptive without making them more suspicious of everyone, more strategic without making them more coercive, and more self-aware without turning ordinary imperfection into a diagnosis.
