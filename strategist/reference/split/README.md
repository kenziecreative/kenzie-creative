# Split

*Break the problem into its component drivers.*

3 frameworks.

- [Driver Tree](driver-tree.md) — A branching diagram that decomposes a problem into its component parts and then into the drivers of each component — so every potential lever is visible before you choose which ones to pull.
- [Bucketing](bucketing.md) — A bottom-up grouping method that starts with a raw list of factors, ideas, or drivers and organizes them into logical clusters — so a messy brainstorm becomes a structured problem map.
- [Hypothesis](hypothesis.md) — A method that converts a problem's branches into explicit, testable propositions — so analysis is directed at confirming or disconfirming specific claims rather than exploring open-endedly.
